Overview

What is AHE Bot?

AHE Bot is a TypeScript-based bot that automatically posts anime images to Bluesky. It fetches images from popular anime image boards, processes them, and posts them with relevant tags.

Built with modern technologies and best practices, it provides a reliable and efficient way to automate content sharing while maintaining strict content filtering.

Key Features
  • Automatic image fetching and processing
  • Smart content filtering system
  • Built-in caching for performance
  • Multiple image provider support
  • Automatic tag processing

Features

Image Processing

Automatic image resizing and optimization for Bluesky's requirements. Supports multiple formats including JPEG, PNG, and GIF.

Content Filtering

Advanced content filtering system with customizable rules and severity levels to ensure appropriate content.

Performance

Built-in caching system and optimized API calls for maximum performance and reliability.

Installation

Prerequisites
  • Node.js (v16 or higher)
  • npm or yarn
  • A Bluesky account
Quick Start
git clone https://github.com/chocoOnEstrogen/ahe-bot.git
cd ahe-bot
npm install
npm run dev
For development, you'll need to set up your environment variables. See the Configuration section for details.

Configuration

Environment Variables
Variable Description Required Default
BLUESKY_IDENTIFIER Your Bluesky handle Yes -
BLUESKY_PASSWORD Your Bluesky app password Yes -
GELBOORU_API_KEY Gelbooru API key No -
PORT Server port number No 3000
Configuration Files

BLUESKY_IDENTIFIER=your.handle
BLUESKY_PASSWORD=your-app-password
GELBOORU_API_KEY=your-api-key
PORT=3000

API Reference

GET /api/providers

Returns a list of available image providers.

Response
{
    "success": true,
    "data": ["gelbooru", "rule34"]
}
GET /api/search

Search for images across providers.

Parameters
Parameter Type Description Required
provider string Provider name (gelbooru, rule34) No
query string Search query No
page number Page number No
Response
{
    "success": true,
    "data": [
        {
            "id": 123456,
            "file_url": "https://example.com/image.jpg",
            "tags": "tag1 tag2 tag3",
            "rating": "safe"
        }
    ],
    "hasMore": true
}

Deployment

Docker Deployment

Deploy using Docker Compose:

docker-compose up -d
PM2 Deployment

Deploy using PM2 process manager:

npm run pm2:start
Manual Deployment

Build and start the application:

npm run build
npm start

Troubleshooting

If you're experiencing authentication issues:

  1. Verify your Bluesky credentials in the .env file
  2. Ensure your app password is correct
  3. Check if your Bluesky account is active

If you're hitting rate limits:

  • Implement proper delay between requests
  • Use the built-in caching system
  • Consider upgrading to a paid API tier
